# 要件定義書 — 掲示板メモアプリ

**プロジェクト名**: BBS Memo（仮称）  
**バージョン**: 1.0.0  
**作成日**: 2026-05-20  
**ステータス**: ドラフト

---

## 1. プロジェクト概要

### 1.1 コンセプト

「スレッドが落ちる」という2ちゃんねる（現5ちゃんねる）型掲示板の文化をメモ・リマインダーに応用したアプリ。通常のメモアプリと異なり、**放置するとスレッドが落ちる**という仕組みを持つ。ユーザーは定期的にスレを更新し続けることで、自然とタスクやメモを忘れずに管理できる。

### 1.2 解決する課題

- 通常のメモ・TODOアプリは「書いたら終わり」になりやすく、見返さない
- リマインダーは通知を無視すれば消えてしまう
- 本アプリは**スレを落とさないために能動的な操作が必要**になる設計とすることで、継続的な注意を促す

### 1.3 対象ユーザー

- 自己管理が苦手でタスクを忘れやすい人
- 2ちゃんねる文化・掲示板文化に親しみのある人
- スマートフォンをメインに使うユーザー

## 3. 機能要件

### 3.1 板（ボード）機能

- ユーザーが任意の板を作成・編集・削除できる
- 板にはジャンル名・説明文を設定できる
- 板ごとにスレ一覧を表示する
- 板の並び順は手動で変更可能
- 板には**タグ（ジャンル）**を複数設定できる（例: `仕事`, `買い物`, `プライベート`）

### 3.2 スレッド機能

#### 3.2.1 スレッドの作成

- スレタイトル（件名）を入力して作成
- 作成時に以下を設定する:
  - **落ち時間**: 最終書き込みから何時間/日後にスレが落ちるか
  - **AIレスの有無**: ON/OFF および返信間隔の設定
  - **タグ**: スレに任意のタグを複数付与できる（板タグを引き継ぐことも可能）
- 作成時に本文（スレ本文 = 1番レス）を記入

#### 3.2.2 スレッドの表示

- 各スレの一覧表示項目:
  - スレタイトル
  - レス数
  - 作成日時 / 最終書き込み日時
  - **落ちるまでの残り時間**（カウントダウン表示）
  - AIレスが有効かどうかのアイコン
  - 添付ファイルありアイコン

#### 3.2.3 スレ一覧の並び替え

スレ一覧は以下のキーで並び替えが可能。選択状態はセッション間で保持する。

| 並び替えキー | 内容 |
|------------|------|
| 作成日時（新しい順） | デフォルト |
| 作成日時（古い順） | |
| 最終書き込み（新しい順） | 最近動いたスレ優先 |
| 残り時間（少ない順） | 要保守スレ優先 |
| レス数（多い順） | 活発なスレ優先 |
| レス数（少ない順） | |
| タイトル（あいうえお順） | |

- スマホUIでは並び替えボタンをタップするとシート（ボトムシート）で選択肢を表示
- PCでは一覧上部にドロップダウンで表示

#### 3.2.3 スレッド落ち

- 条件: **最終書き込みから設定した時間が経過、または最大レス数（1000レス）に到達すると自動的にスレが落ちる**
- 落ち時間はスレ作成時に設定する。設定値の選択肢:

| 選択肢 | 内容 |
|--------|------|
| 超短期 | 5分 / 15分 / 30分 |
| 短期 | 1時間 / 3時間 / 6時間 |
| 中期 | 12時間 / 1日 / 3日 |
| 長期 | 7日 / 14日 / 30日 |

- スレが落ちると **アーカイブ化**され、以後の書き込みは不可
- 落ち前には残り時間を視覚的に強調表示（赤文字・点滅など）
- スレが落ちても閲覧は可能（読み専用）
- **次スレ作成機能（1000レス到達時）**:
  - 1000レスに到達してスレ落ちした場合、最終レス（1000番）の下部に「次スレを作成」ボタンを表示する。
  - ボタン押下により、元のスレの設定（落ち時間、AI設定、タグ、所属する板）を引き継ぎ、タイトル末尾のナンバリングをインクリメント（例: 「会議メモ」→「会議メモ 2」、「会議メモ 2」→「会議メモ 3」）した状態で、自動的に新しいスレッドを作成し遷移する。

#### 3.2.4 スレの保守（age / sage）

- レスを書き込むことでスレの落ち時間がリセットされる（デフォルト）
- `sage` オプションを指定した場合はリセットされない（落ち時間の消費が続く）
- スレを書き込まずに落ち時間を延長する「age保守」ボタンを設ける（任意機能）

### 3.3 レス（投稿）機能

#### 3.3.1 投稿フォーム

| 項目 | 内容 |
|------|------|
| 名前欄 | 任意入力。空白時は「名無しさん」 |
| メール欄 | 任意入力。`sage` と入力でsage投稿 |
| 本文欄 | 必須。最大2000文字 |
| ファイル添付欄 | 任意。後述の仕様に従う |
| 送信ボタン | 「書き込む」 |

#### 3.3.2 ファイル・画像添付機能

##### 対応ファイル形式

| 種別 | 対応形式 |
|------|---------|
| 画像 | JPEG / PNG / GIF / WebP / HEIC |
| ドキュメント | PDF / TXT / MD |
| その他 | 制限なし（ただし上限サイズあり） |

- 1レスあたりの添付ファイル数: 最大 **4ファイル**
- 1ファイルの上限サイズ: **10MB**（圧縮前）
- ファイルは **IndexedDB に Base64 エンコードして保存**

##### 画像圧縮機能

- 添付画像に対して圧縮処理を適用するかどうかを選択可能
- 圧縮はブラウザ内で処理（`Canvas API` を使用、外部送信なし）
- 圧縮設定の選択肢:

| 設定 | 内容 |
|------|------|
| 圧縮なし | オリジナルサイズで保存 |
| 軽め（80%品質） | ファイルサイズ小、画質ほぼ維持 |
| 標準（60%品質） | バランス型（デフォルト） |
| 強め（40%品質） | ファイルサイズ最小 |
| カスタム | 品質・最大解像度を手動設定 |

- 圧縮後のファイルサイズを添付前にプレビュー表示（例: `1.2MB → 320KB`）

##### AA（アスキーアート）変換機能

- 画像添付時に「AA変換」オプションが表示される
- ボタンを押すと Canvas API でモノクロ変換 → 文字マッピング処理し、AA文字列を生成
- **プレビューダイアログ**でAA変換結果を確認してから、本文挿入 or キャンセルを選択できる
- AA変換のスタイル選択:

| スタイル | 使用文字 |
|--------|---------|
| ドット系 | `░ ▒ ▓ █` |
| 記号系 | `. : ; i l I` など |
| カスタム | 使用文字を手動入力 |

- 変換解像度（横幅の文字数）を 20 / 40 / 80 文字から選択可能
- プレビューはモーダルで表示し、AA文字フォント（等幅）でリアルタイム確認

##### ファイル表示

- 画像はレス内にサムネイル表示（タップで拡大）
- 画像以外のファイルはファイル名・サイズ付きのダウンロードリンクとして表示
- アーカイブ済みスレのファイルも引き続き閲覧・ダウンロード可能

#### 3.3.2 レス表示項目

各レスには以下の情報を表示する:

- レス番号（例: `1`、`2`、`3`…）
- 名前（名無しさん or 入力した名前）
- 日時（例: `2026/05/20(水) 14:32:15.21`）
- ID（セッション単位のランダムID、例: `ID:Ab3Xk9mP`）
- 本文

#### 3.3.3 アンカー（安価）機能

- `>>番号` の形式で特定レスへのリンクを生成（例: `>>3`）
- アンカー部分はクリック/タップでそのレスにジャンプ
- アンカーされたレスはハイライト表示
- スマホでは長押しでレス番号を引用（`>>N` を入力欄に挿入）

#### 3.3.4 その他テキスト機能

| 記法 | 表示 |
|------|------|
| `>引用テキスト` | 緑文字の引用表示 |
| `>>N` | アンカー（リンク） |
| `http://...` | URLの自動リンク化 |
| `■■■` など | AA（アスキーアート）対応フォント適用 |

### 3.4 AI レス機能

#### 3.4.1 概要

各スレッドに対してAI（主要APIに対応する）が自動でレスを返す機能。

#### 3.4.2 設定項目

| 設定 | 内容 |
|------|------|
| 有効/無効 | スレ作成時またはスレ設定画面で切替 |
| 返信間隔 | 下記の選択肢から選ぶ |
| AIの名前 | 任意設定（デフォルト: `AI◆bot`） |
| AIの性格/口調 | プリセット選択 or 自由入力 |
| スレ落ちへの影響 | 設定で変更可能（後述） |

#### 3.4.3 返信間隔の選択肢

| 選択肢 | 内容 |
|--------|------|
| 数分 | 2〜10分のランダム |
| 数十分 | 10〜60分のランダム |
| 数時間 | 1〜6時間のランダム |
| 数日 | 1〜3日のランダム |
| ランダム | 数分〜数日の中からランダム |
| カスタム | 分単位で手動入力 |

#### 3.4.4 スレ落ちへの影響設定

AIレスが書き込まれた場合、スレの落ち時間リセットへの影響を以下から選択可能:

| 設定値 | 動作 |
|--------|------|
| 通常レスと同じ（デフォルト） | AIレスで落ち時間がリセットされる |
| sage扱い | AIレスでは落ち時間がリセットされない |
| カスタム | AI専用の影響度を設定 |

#### 3.4.5 主要API対応とAPIキーの管理

- 以下の主要AI APIおよび互換APIに対応する：
  - **Google Gemini API** (モデル例: `gemini-2.5-flash`, `gemini-2.0-flash` など)
  - **Anthropic Claude API** (モデル例: `claude-3-5-sonnet`, `claude-3-5-haiku` など)
  - **OpenAI GPT API** (モデル例: `gpt-4o`, `gpt-4o-mini` など)
  - **OpenAI互換 API** (DeepSeek, Groq, Ollama, LM Studio などのローカル/外部 LLM サービス)
- モデル名取得および選択機能：
  - 各APIのエンドポイント（例: `/v1/models` や Gemini/Anthropic のモデル一覧API）を呼び出し、利用可能なモデル一覧を動的に取得して選択できるUIを搭載する。
  - 手動でのモデル名入力（自由入力）にも対応する。
- 各APIキーおよび使用するプロバイダ/モデル設定は、アプリ設定画面でユーザーが入力する。
- APIキーは **IndexedDB にローカル保存**される（外部サーバー送信はなく、ブラウザから直接APIをコールする。CORS対応のため、必要に応じてカスタムエンドポイント/プロキシ設定も可能とする）。
- 使用するプロバイダのAPIキーが未設定の場合は、そのプロバイダを利用したAI機能を無効化し、設定を促すダイアログを表示する。

#### 3.4.6 AIの動作

- スレッドの全レス履歴をコンテキストとしてAPIに送信
- AIは掲示板の住民として自然な口調でレスを返す
- 返信先アンカー（`>>N`）を使ってレスをつなぐ

### 3.5 アーカイブ機能

- 落ちたスレは自動的にアーカイブに移動
- アーカイブ一覧ページで閲覧可能
- アーカイブされたスレへの書き込みは不可（読み専用）
- アーカイブスレにはUIで「【落ちました】」などのバッジを表示
- アーカイブは手動削除可能

### 3.7 タグ・ジャンル絞り込み機能

#### 3.7.1 タグの仕様

- タグは板・スレの両方に付与可能（自由入力 + 既存タグからの選択）
- タグはアプリ全体で共通管理。タグ一覧画面から編集・削除できる
- 1つの板またはスレに設定できるタグ数: 最大 **10個**
- タグ名の最大文字数: **20文字**

#### 3.7.2 タグによる絞り込み

- スレ一覧画面の上部にタグフィルターバーを表示
- 複数タグの AND / OR 絞り込みに対応
- 絞り込み状態は画面遷移後も保持（設定でOFF可能）
- スマホではタグフィルターバーを横スクロール式のチップUIで表示

#### 3.7.3 タグ一覧・管理

- 設定画面または専用ページからタグ一覧を確認・編集・削除できる
- 各タグに色を設定でき、スレ一覧・スレッドヘッダーにカラーバッジで表示
- 未使用タグの一括削除機能あり

### 3.8 検索機能

#### 3.8.1 検索の種類

| 検索種別 | 対象 | 説明 |
|--------|------|------|
| **全文検索** | 全スレ・全レスの本文 | アプリ内の全テキストを横断検索 |
| **スレタイ検索** | スレタイトルのみ | 高速・軽量な部分一致検索 |
| **スレ内検索** | 開いているスレのレス本文 | スレッド閲覧中に本文を絞り込む |
| **ファイル名検索** | 添付ファイルのファイル名 | 添付ファイルを含むレスを検索 |
| **投稿者名検索** | レスの名前欄 | 特定の名前のレスを絞り込む |
| **ID検索** | レスのID | 同一IDのレスをまとめて表示 |

#### 3.8.2 検索 UI

- グローバル検索バーはヘッダーまたはボトムナビから常時アクセス可能
- スレッド内検索はスレ画面上部の🔍アイコンから起動（ブラウザのCtrl+Fに相当）
- 検索結果はレス単位でカード表示し、スレタイ・投稿日時・マッチ箇所（前後の文脈付き）を表示
- 検索キーワードはヒット箇所をハイライト表示

#### 3.8.3 絞り込み・フィルター

検索結果に対して以下の条件で絞り込みが可能:

| フィルター | 内容 |
|----------|------|
| 板の指定 | 特定の板に検索対象を限定 |
| タグの指定 | 特定タグのスレに限定 |
| スレのステータス | アクティブ / アーカイブ / 両方 |
| 期間指定 | 投稿日時の範囲（開始日〜終了日） |
| 添付ファイルあり | 添付ファイルを含むレスのみ |
| AIレスを除く | AIが投稿したレスを除外 |

#### 3.8.4 検索のパフォーマンス要件

- スレタイ検索・スレ内検索: **即時（タイプに追従してリアルタイム検索）**
- 全文検索: 入力確定（Enterまたは検索ボタン）後に実行
- 全文検索の目標レスポンス: **1秒以内**（レス数1万件以下の環境）
- IndexedDB のインデックスを活用し、全スキャンを避ける実装とする

#### 3.8.5 検索履歴

- 直近10件の検索キーワードを保存・表示（検索バーフォーカス時に一覧表示）
- 検索履歴は設定画面から削除可能

### 3.6 設定機能

設定画面はカテゴリ別にセクション分けして表示する。

#### 3.6.1 AI設定

| 設定項目 | 内容 |
|---------|------|
| AIプロバイダ選択 | Gemini / Anthropic / OpenAI / OpenAI互換 から選択 |
| APIキー設定 | 各プロバイダ（Gemini, Anthropic, OpenAI, OpenAI互換）のキー入力・保存・マスク表示・削除 |
| モデル選択・取得 | ボタン押下で動的に利用可能なモデル一覧を取得し、ドロップダウンから選択（手動入力も可） |
| カスタムエンドポイント | CORS制限やプロキシ、OpenAI互換APIを使用する場合のAPIベースURL（任意） |
| デフォルトAI名前 | 全スレ共通の初期AI名（例: `AI◆bot`） |
| デフォルトAI性格 | プリセット選択 or 自由入力テキスト |
| デフォルト返信間隔 | 新規スレ作成時の初期値 |
| AIレスのスレ落ち影響 | デフォルト設定（通常 / sage / カスタム） |

#### 3.6.2 投稿・スレ設定

| 設定項目 | 内容 |
|---------|------|
| デフォルト名前 | レス投稿時の初期名前 |
| デフォルト落ち時間 | 新規スレ作成時の初期値 |
| デフォルト画像圧縮 | 添付時の初期圧縮設定 |
| スレの最大レス数 | 1000レス到達で自動スレ落ち（固定仕様） |

#### 3.6.3 表示設定

| 設定項目 | 内容 |
|---------|------|
| テーマ | ノーマル / ダーク / ピンク / ギーク |
| フォントサイズ | 小（13px） / 中（15px） / 大（17px） |
| AAフォント | AA描画に使う等幅フォント名（デフォルト: `Mona` / `MS PGothic`） |
| 残り時間の警告閾値 | 何%以下になったら赤く警告するか（例: 20%） |
| スレ一覧のデフォルト並び順 | 並び替えキーの初期値 |
| 要保守スレ優先表示 | ON/OFF（残り時間が少ないスレを上に固定） |

#### 3.6.4 通知設定

| 設定項目 | 内容 |
|---------|------|
| ブラウザ通知 | ON/OFF（要通知権限） |
| 通知タイミング | スレ落ちの何分前に通知するか（例: 30分前・1時間前） |
| 通知するスレの条件 | 全スレ / 通知ONのスレのみ |

#### 3.6.5 データ管理

| 操作 | 内容 |
|------|------|
| エクスポート（全データ） | 全板・スレ・レス・ファイル・設定を JSON + 添付ファイルの ZIP でエクスポート |
| エクスポート（スレ単位） | 指定スレのレスとファイルのみエクスポート |
| インポート | ZIP または JSON ファイルを選択してデータ復元。既存データとのマージ or 上書きを選択 |
| ストレージ使用量確認 | 詳細は下記参照 |
| アーカイブの一括削除 | 全アーカイブスレを削除（確認ダイアログ付き） |
| 全データ削除 | 全データ初期化（二重確認ダイアログ付き） |
| 検索履歴の削除 | 保存された検索キーワード履歴をクリア |

##### ストレージ使用量ダッシュボード

設定画面のデータ管理セクションに専用のストレージ使用量パネルを設ける。

**表示項目:**

| 項目 | 内容 |
|------|------|
| 総使用量 / 概算上限 | 例: `124MB / 約500MB`（プログレスバー付き） |
| カテゴリ別内訳 | テキストデータ / 添付ファイル / アーカイブ / 設定 の割合を円グラフまたは積み上げバーで表示 |
| 板・スレ別使用量 | 容量が多い順にリスト表示（上位10件） |
| 使用量の警告 | 上限の80%超過で黄色、90%超過で赤色の警告バナーを表示 |
| 最終計測日時 | ストレージ計測は `navigator.storage.estimate()` を使用 |

**操作:**

- 「今すぐ再計測」ボタン（手動更新）
- 容量が大きいスレをタップして即座に削除ページへジャンプ
- アーカイブ一括削除・添付ファイル一括削除のショートカットリンクをパネル内に配置

**自動警告:**

- アプリ起動時にストレージ使用量が80%を超えていた場合、バナー通知を表示
- 通知には「アーカイブを削除する」「設定を開く」のクイックアクションボタンを設ける

##### エクスポート形式の詳細

```
export_20260520.zip
├── data.json          # 全板・スレ・レス・設定のメタデータ
└── files/
    ├── {post_id}_{filename}   # 添付ファイル（画像など）
    └── ...
```

##### インポートの動作

- `data.json` のスキーマバージョンを確認し、互換性チェックを実施
- 既存データとのID衝突が発生した場合:
  - **マージモード**: 新しい方のタイムスタンプを優先
  - **上書きモード**: インポートデータで既存を完全置換
- インポート前にバックアップを自動作成（インポート操作の前にエクスポートを促すダイアログを表示）

## 5. 非機能要件

### 5.1 パフォーマンス

- 初回ロード: 3秒以内（4G環境）
- スレ表示: 1秒以内
- IndexedDB の読み書き: 500ms 以内

### 5.2 オフライン動作

- AI機能を除いた全機能はオフライン（ネットワーク不要）で動作すること
- Service Worker による PWA 対応（任意・将来対応）

### 5.3 データ容量

- IndexedDB の上限: ブラウザによるが通常は数十〜数百MB
- スレッドのアーカイブが大量になった場合の警告表示

### 5.4 セキュリティ

- API キーはローカルにのみ保存し、外部に送信しない
- XSS 対策: ユーザー入力は全てエスケープして表示

## 6. 決定事項・基本仕様の確定

| # | 項目 | 確定された仕様・方針 |
|---|------|---------------------|
| 1 | スレの復活機能 | **なし**。一度落ちた（期限切れ・1000レス到達）スレは完全にアーカイブ化され、閲覧専用（読み込み専用）となる。 |
| 2 | スレの最大レス数 | **1000レス固定**。1000レスに到達した時点で自動的にスレが落ち、アーカイブに移動する。 |
| 3 | NG機能 | **実装する**。レス内の特定ワードを非表示にする「NGワード」および、特定IDの長押しなどから登録可能な「NGID」を実装。NG対象は「あぼーん」と表示され、タップで一時的に展開可能とする。 |
| 4 | 検索機能 | **実装する**。IndexedDBのインデックスやメモリ内フィルタを活用し、全文検索、スレタイ検索、スレ内検索、ファイル名検索、投稿者名検索、ID検索を全て実装する。 |
| 5 | ファイル添付の上限 | **1レスあたり最大4ファイル、合計15MB**まで添付可能とする。画像についてはCanvas APIによる圧縮処理を施すことで容量を削減できる。 |
| 6 | AA変換の精度 | **Canvas APIによる自作の文字マッピングロジックで実装**。外部ライブラリを介さず、無料で動作し、等幅フォント環境でいい感じに表示されるAAに変換する。 |
